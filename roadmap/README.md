# NexusLink Beginner Roadmap

This roadmap breaks the project into small, learn-as-you-build phases.
It is designed for a beginner who wants to complete the full project end to end without burnout.

## How To Use This Folder

- Follow phases in order from `1` to `7`.
- Do not move to the next phase until the current phase "exit criteria" is complete.
- Keep daily sessions short (60-120 minutes).
- Track progress using the weekly template in `templates/week-plan.md`.
- When stuck, reduce scope for the day instead of quitting the phase.

## Time Strategy (Beginner Friendly)

- Recommended pace: 8-12 hours/week.
- Total timeline: around 12-18 weeks.
- Rule: "One core concept + one coding goal" per day.

7. `phases/07-polish-and-advanced-extras.md`

## Burnout Prevention Rules

- Keep "must do" list to 3 tasks/day max.
- Every 7th day is review day (no new coding).
- If blocked for more than 90 minutes, document blocker and switch to a smaller task.
- Celebrate phase completions with a short retrospective note.

## Output You Should Have At The End

- A complete URL shortener with redirect + analytics.
- Measured performance notes and benchmark results.
- Docs and architecture decisions you can explain in interviews.
- A portfolio-grade project executed step by step.
